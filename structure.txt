================================================================================
  Architecture
================================================================================

── Data Flow ──────────────────────────────────────────────────────────────────
  UI (board.js/sidebar.js) → api.js → window.__TAURI__.core.invoke()
       → Rust commands (commands.rs) → Box<dyn Storage> trait methods
       → SqliteStorage (db/sqlite.rs) → SQLite via rusqlite

── Storage Trait (storage.rs) ─────────────────────────────────────────────────
  All 20 backend commands work through `Box<dyn Storage>`. To swap backends:
    1. Implement Storage trait for new backend
    2. Change one line in main.rs
  No other code changes needed.

── Database Schema (SQLite, v2) ───────────────────────────────────────────────
  schema_version  (version INTEGER)
  folders         (id INTEGER PK, name TEXT, parent_id INTEGER, created_at TEXT)
  projects        (id INTEGER PK, name TEXT, desc TEXT, folder_id INTEGER FK,
                   created_at TEXT)
  nodes           (id INTEGER PK, project_id INTEGER FK, title TEXT, desc TEXT,
                   files TEXT/JSON, timeline TEXT/JSON, created_at TEXT)

── Frontend Module Dependency Graph ───────────────────────────────────────────
  main.js ─→ store.js, api.js, utils.js, theme.js, sidebar.js, board.js,
             modals.js, chat.js, preview.js
  sidebar.js ─→ store.js, api.js, utils.js
  board.js ─→ store.js, api.js, utils.js, preview.js, chat.js
  modals.js ─→ store.js, api.js, utils.js
  chat.js ─→ store.js, api.js, utils.js
  preview.js ─→ store.js, api.js, utils.js
  Zero circular imports at module init time.

── CSS Architecture (main.css, optimized) ─────────────────────────────────────
  Custom properties: --glass-heavy, --glass-medium, --focus-ring, --hover-lift
  Button system: base .btn + [data-variant="primary|ghost|danger|ghost-danger"]
  Naming: .modal.active / .chat-panel.active (unified)
  13 sections: Variables → Reset → Header → Buttons → Layout → Sidebar →
  Board → Modals → Upload → Stats → Chat → Search → Scrollbar

── Testing ────────────────────────────────────────────────────────────────────
  Backend:  35 integration tests (cargo test) — in-memory SQLite, covers all CRUD
  Frontend: 22 utils assertions (open tauri-app/tests/test.html in browser)

================================================================================
  Key Dependencies
================================================================================

  Rust:   tauri 2, rusqlite 0.32 (bundled SQLite), serde/serde_json, chrono,
          ureq 2 (native-tls + json), base64 0.22, zip 0.6
  JS:     zero npm dependencies — ES modules, native WebView APIs
